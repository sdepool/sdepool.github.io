Hexo初始安装

软件安装

  1. 安装node.js，勾选Add to PATH选项。
  2. 安装Git。

Github 配置

Github网站设置

  1. 新建gitpage（resposery），项目必须要遵守以下格式：账户名.github.io,如 sdepool.github.io，并且设成public，hexo生成的网页文件会部署会在sdepool.github.io中的master分支中，详见Hexo设置。
  2. 建立sdepool.github.io分支如：githexo-dev,并设置其为默认分支。此分支用来存储hexo源代码， 新建sdblog-dev（resposery），可设其为private，用来存储hexo源配置文件。

Git-submodule相关

删除子模块（子仓库）

  1. 逆初始化模块，其中{MOD_NAME}为模块目录，执行后可发现模块目录被清空：

    但目录还在，最后需要把目录删除。

  2. 删除.gitmodules中记录的模块信息（–cached选项清除.git/modules中的缓存，但文件夹还是存在，需要手动删除）

  3. 提交更改到代码库，可观察到’.gitmodules’内容发生变更(没有作用？)

  4. 删除gitsubmodule里相关部分（以下步骤做完后才能再增加子模块）:

    • 删除.git/config文件里相关字段
    • 删除子仓库目录
    • 删除.git/module里的子仓库目录
